# 项目术语表 (Glossary)

> Nova AI Gateway 项目核心概念与术语的统一定义。
> 所有文档、代码、沟通中应使用本术语表保持一致。

Version: v1.0
Status: Active
Owner: AI Project Manager
Last Updated: 2026-07-20

---

## A

### ADR (Architecture Decision Record)
架构决策记录。记录每个重要技术决策的上下文、方案对比、选择理由和影响。存放在 `docs/ADR/` 目录。

### Agent Runtime
AI Agent 运行时环境。支持 Tool Calling、Memory、Knowledge Base 等能力的执行引擎。（Phase 5 开发）

### API Gateway (Gateway)
系统的核心代理层。接收用户请求，转发到 Router Engine，最终调用 Provider 返回结果。性能目标：主链路 < 10ms。

### API Key
用户访问 Gateway 的凭证。格式 `sk-campus-xxxxx`，每个用户可创建多个，支持权限控制和用量追踪。

### Auth Service
认证服务。负责用户登录注册、JWT 签发、API Key 校验、RBAC 权限控制。

---

## B

### Billing Service
计费服务。管理套餐、订单、充值、退款、发票。（商业化模块，Phase 6 重点）

### Budget
预算管理。按部门/项目设置预算上限，超过阈值触发预警或自动限制。

---

## C

### CEO (产品负责人)
AI Company 中的最终决策者（即你）。负责产品方向、商业模式、Roadmap、Standards 审批。

### Cost Engine
成本引擎。管理各 Provider 的采购价、售价、利润分析，提供成本优化建议。

---

## D

### Dashboard
运营仪表盘。展示今日请求数、Token 消耗、收入/成本/利润、在线用户、QPS 等实时数据。

### Decision Center
AI Company 的决策中心。所有需求必须先经过需求分析、规模评估、影响分析，才能进入开发流程。

---

## E

### Event Queue
事件队列。用于异步处理日志、成本、统计等非关键路径数据，避免拖慢主链路。

---

## F

### Full Stack Engineer
AI Company 中的全栈工程师角色。MVP 阶段不拆分前后端，统一由该角色负责实现。技术栈：Go + Vue3。

---

## G

### Gateway (API Gateway)
见 API Gateway。

### Glossary
术语表。即本文档，用于统一项目概念定义。

---

## L

### Log Engine
日志引擎。记录每一次请求的完整信息（时间、用户、模型、Provider、Token、耗时、状态等），支持 Request/Response 原文查看。

---

## M

### Model (逻辑模型)
业务层面的模型概念。如 GPT5、Claude Sonnet、DeepSeek V3。一个 Model 可以绑定多个 Provider。

### MVP (Minimum Viable Product)
最小可运行产品。项目开发的核心原则：优先完成可用版本，禁止过度设计。

---

## P

### Phase
项目阶段。当前路线图：P0(初始化) → P1(MVP) → P2(Policy) → P3(Developer) → P4(Skill) → P5(Agent) → P6(Enterprise)。

### Policy Engine
策略引擎。系统的决策中心，负责 Cost Engine 决策、智能路由、配额限制、预算管控、预警触发。

### Pricing
定价中心。配置各模型的售价策略（固定价格、成本倍数、最低利润率）。

### Product Manager (PM)
AI Company 中的产品经理角色。负责 PRD、用户故事、验收标准。

### Provider (供应商)
真实的大模型供应商。如 OpenAI、Anthropic、DeepSeek、Google Gemini、阿里 Qwen。

### Provider Adapter
供应商适配层。统一不同 Provider 的 API 差异，提供一致的内部调用接口。

---

## Q

### QA (Quality Assurance)
AI Company 中的测试工程师角色。负责功能测试、回归测试、API 测试、验收测试。

### Quota (配额)
用户/组织的使用限制。支持限制 Token 数、请求次数、图片数、音频时长等维度。

---

## R

### Requirement Analyzer
AI Company 中的需求分析师角色。负责需求类型判断、规模评估(S0~S4)、风险分析、Workflow 推荐。

### Reviewer
AI Company 中的代码评审角色。负责 Code Review、架构评审、文档评审。

### Router / Router Engine
路由引擎。支持多种路由策略：最低价格、最快速度、权重分配、轮询、失败切换。

---

## S

### S0 ~ S4 (需求规模等级)
| 等级 | 类型 | 工作量 |
|:----:|------|--------|
| S0 | 微小修改 | < 2h |
| S1 | 普通功能 | 0.5~2天 |
| S2 | 模块开发 | 2~7天 |
| S3 | 系统升级 | 1~4周 |
| S4 | 战略项目 | > 1个月 |

### Skill Marketplace
技能市场。用户可发布、安装、收费的 AI Skill 平台。（Phase 4 开发）

### Standards
AI Company 的标准规范体系。共 11 项标准文件，存放在 `ai-company/01-standards/`。

---

## T

### Token
大模型的输入/输出计量单位。用于计费、配额限制、用量统计的核心指标。

---

## W

### Workflow
AI Company 中定义的标准化开发流程。根据需求类型和规模匹配不同的 Workflow（如 S2 Feature: PM → Architect → Engineer → Reviewer → QA）。
