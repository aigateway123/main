## 四、初始化步骤

### Step 1：进入项目目录

```bash
cd "/Users/fuxiansheng/Desktop/AI Gateway/aigateway"
```

### Step 2：准备本地环境变量

复制模板文件，生成本地开发配置：

```bash
cp .env.example .env.development
```

建议优先修改以下配置：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `POSTGRES_PASSWORD` | `postgres` | 本地数据库密码 |
| `JWT_SECRET` | `change-me-in-production` | JWT 签名密钥 |
| `OPENAI_API_KEY` | 空 | OpenAI Key |
| `CLAUDE_API_KEY` | 空 | Claude Key |
| `DEEPSEEK_API_KEY` | 空 | DeepSeek Key |
| `QWEN_API_KEY` | 空 | Qwen Key |

### Step 3：启动基础设施

在项目根目录执行：

```bash
docker compose -f infra/docker/docker-compose.yml up -d
```

启动后会创建两个容器：
- `nova-ai-gateway-postgres`
- `nova-ai-gateway-redis`

### Step 4：检查容器状态

```bash
docker compose -f infra/docker/docker-compose.yml ps
```

预期结果：
- `postgres` 为 `healthy`
- `redis` 为 `healthy`

### Step 5：验证连接信息

本地默认连接如下：

| 服务 | 地址 |
|------|------|
| PostgreSQL | `localhost:5432` |
| Redis | `localhost:6379` |

数据库默认信息：

| 项目 | 默认值 |
|------|--------|
| Database | `nova_ai_gateway` |
| Username | `postgres` |
| Password | `postgres` |

## 五、环境变量说明

`.env.example` 当前包含 6 类配置：

| 分类 | 说明 |
|------|------|
| 应用基础配置 | `APP_ENV`、`APP_PORT`、`LOG_LEVEL` |
| PostgreSQL | `POSTGRES_*`、`DATABASE_URL` |
| Redis | `REDIS_*`、`REDIS_ADDR` |
| 认证安全 | `JWT_SECRET`、`JWT_EXPIRES_IN` |
| 服务端口 | `GATEWAY_PORT`、`POLICY_ENGINE_PORT` 等 |
| Provider 配置 | OpenAI / Claude / Gemini / DeepSeek / Qwen |

说明：
- `.env.example` 可以提交到仓库
- `.env.development`、`.env.test`、`.env.production` 禁止提交
- 生产环境必须替换默认密钥与密码

## 六、常见问题

### 1. Docker 未启动

表现：
- `docker compose` 报错
- 容器无法拉起

处理：
- 确认 Docker Desktop 已启动
- 重试 `docker compose -f infra/docker/docker-compose.yml up -d`

### 2. 5432 或 6379 端口被占用

处理方式：
- 停掉本机已有 PostgreSQL / Redis
- 或修改 `.env.development` 中的端口映射

### 3. Provider API Key 为空

影响：
- 本地基础设施仍可正常启动
- 但无法调用对应大模型 Provider

处理：
- 在 `.env.development` 中填入实际 API Key
