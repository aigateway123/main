# 用户画像

> 基于 Nova AI Gateway 的产品定位，梳理核心目标用户群体。

---

## 一、用户分层总览

```
                  ┌─────────────────────┐
                  │   Enterprise 客户    │  ← 高客单价，深度定制
                  │   (签约+私有部署)     │
                  └─────────────────────┘
                           ↑
                  ┌─────────────────────┐
                  │   Pro 开发者/团队    │  ← 核心付费用户
                  │   (按月/按量付费)    │
                  └─────────────────────┘
                           ↑
                  ┌─────────────────────┐
                  │   Free 个人开发者    │  ← 流量入口、口碑传播
                  │   (免费额度引流)     │
                  └─────────────────────┘
```

---

## 二、Persona 1：个人开发者（Free 用户）

| 维度 | 描述 |
|------|------|
| **典型身份** | 独立开发者、在校学生、AI 技术爱好者 |
| **技术能力** | 有 API 调用经验，了解 OpenAI 兼容接口 |
| **核心痛点** | 多个模型各自注册、充值、管理 Key 太麻烦；API 格式不统一 |
| **使用场景** | 个人项目调用 GPT/Claude/DeepSeek 做原型验证、AI 应用开发 |
| **付费意愿** | 低，但愿意为便利付费 |
| **月预算** | ¥0 ~ ¥100 |
| **关键需求** | 统一的 OpenAI 兼容 API、免费额度、模型选择自由 |
| **获取渠道** | GitHub、开发者社区、技术博客 |

---

## 三、Persona 2：创业团队 / 小型公司（Pro 用户）

| 维度 | 描述 |
|------|------|
| **典型身份** | CTO、技术负责人、全栈工程师 |
| **团队规模** | 2~20 人 |
| **技术能力** | 强，会自建基础设施，但希望专注业务 |
| **核心痛点** | API 成本不可控；单 Provider 故障影响业务；多项目 Key 管理混乱 |
| **使用场景** | 产品集成 AI 能力（客服、写作、代码生成、数据分析等） |
| **付费意愿** | 中等，ROI 清晰就愿意付费 |
| **月预算** | ¥100 ~ ¥5,000 |
| **关键需求** | 成本可视化、用量统计、智能路由、API Key 管理、日志排查 |
| **获取渠道** | 技术社区、Google 搜索、产品推荐 |

---

## 四、Persona 3：企业客户（Enterprise 用户）

| 维度 | 描述 |
|------|------|
| **典型身份** | 技术VP、架构师、基础设施负责人 |
| **团队规模** | 50~500+ 人 |
| **技术能力** | 强，有独立基础设施团队 |
| **核心痛点** | 安全合规（数据不出境）、多部门预算分摊、审计需求、SLA 保障 |
| **使用场景** | 全公司 AI 能力统一接入、内部知识库、客服系统、代码辅助 |
| **付费意愿** | 高，关注总拥有成本和稳定性 |
| **月预算** | ¥5,000 ~ ¥100,000+ |
| **关键需求** | 私有部署、SSO、审计日志、多租户、预算管控、SLA 99.9% |
| **获取渠道** | 销售团队、行业会议、合作伙伴推荐 |

## 五、用户规模预估（MVP 阶段）

| 用户类型 | 预估数量 | 月收入预估 |
|---------|---------|-----------|
| Free 开发者 | 1,000 ~ 5,000 | ¥0 |
| Pro 团队 | 100 ~ 500 | ¥10,000 ~ ¥50,000 |
| Enterprise | 5 ~ 20 | ¥25,000 ~ ¥200,000 |
| **合计** | **1,105 ~ 5,520** | **¥35,000 ~ ¥250,000** |

> 以上为 MVP 上线后 3~6 个月的保守预估。

---

## 六、各阶段聚焦的用户群

| Phase | 聚焦用户 | 目标 |
|-------|---------|------|
| P1 (MVP) | Free + Pro 开发者 | 验证产品价值，积累种子用户 |
| P2 (Policy) | Pro 团队 | 成本优化能力驱动付费转化 |
| P3 (Developer) | Pro + Enterprise | 开发者体验驱动用户增长 |
| P4~P6 | 全量用户 | 商业化规模化 |
