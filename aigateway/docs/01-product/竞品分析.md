# 竞品分析

> Nova AI Gateway 的市场竞争格局分析。

---

## 一、市场定位

Nova AI Gateway 定位为 **面向企业和 AI 开发者的大模型统一接入平台**，核心差异化在于：

1. **智能路由 + 成本优化** — 不只是 Gateway，更是 Policy Engine
2. **自建可控** — Docker 私有部署，数据不出境
3. **全栈能力** — 从 Gateway 到 Agent Runtime 的完整平台

---

## 二、竞品矩阵

### 2.1 直接竞品

| 产品 | 类型 | 开源 | 目标用户 | 核心能力 | 弱点 |
|------|------|------|---------|---------|------|
| **Nova AI Gateway** | 统一接入平台 | ❌ (自建) | 开发者+企业 | Gateway + Router + Cost Engine + Agent Runtime | 起步晚、生态弱 |
| **One API** | API 聚合 | ✅ 开源 | 开发者 | 多模型统一 API | 无成本优化、无管理后台、运维靠手动 |
| **LiteLLM** | SDK 代理 | ✅ 开源 | Python 开发者 | SDK 级多模型调用 | 仅限 Python、无可视化、企业功能弱 |
| **Portkey** | AI Gateway | ❌ SaaS | 开发者+企业 | Gateway + 可观测性 + 成本 | SaaS 不能私有部署，价格高 |
| **Kong AI Gateway** | API Gateway 插件 | ✅ 开源 | 企业 | 基于 Kong 的 AI 流量管理 | 配置复杂，非 AI 原生产品 |
| **APISIX AI Plugin** | API Gateway 插件 | ✅ 开源 | 企业 | 基于 APISIX 的 AI 代理 | 功能基础，需要二次开发 |

### 2.2 间接竞品

| 产品 | 类型 | 说明 |
|------|------|------|
| **OpenRouter** | 模型市场 | 提供统一 API + 模型选择，SaaS 模式，不能自建 |
| **Azure OpenAI** | 云服务 | 只支持 OpenAI 系模型，绑定 Azure 生态 |
| **AWS Bedrock** | 云服务 | 只支持 AWS 生态模型，企业定制化有限 |
| **LangSmith / LangFuse** | 可观测性 | 侧重 LLM 应用观测，非 API Gateway |

## 四、差异化优势分析

### Nova AI Gateway 的护城河

```
竞品做的是「API 聚合」—— 把多个模型接到同一个接口。
Nova 做的是「AI Infrastructure」—— 从接入、路由、策略到 Agent 的全栈平台。
```

| 优势 | 说明 | 竞品对比 |
|------|------|---------|
| **Cost Engine** | 实时成本分析、利润计算、优化建议 | Portkey 有类似但贵；One API 无 |
| **Policy Engine** | 基于成本的智能路由、配额、预算、预警 | 唯一具备此能力的独立产品 |
| **全栈能力** | Gateway + Router + Policy + Agent Runtime | 竞品只覆盖单点 |
| **私有部署** | Docker Compose 即可部署 | Portkey 不能私有；Kong 配置复杂 |
| **价格优势** | 自建无 SaaS 加价，按需付费 | Portkey 加价 10~30% |

### Nova AI Gateway 的劣势

| 劣势 | 说明 | 应对策略 |
|------|------|---------|
| **生态起步晚** | 无社区、无 SDK、插件少 | MVP 优先 OpenAI 兼容 API 降低迁移成本 |
| **品牌认知度低** | 新品牌无人知晓 | 开源部分能力、技术博客引流 |
| **企业信任需要时间** | 无客户案例 | 初期聚焦小型团队，积累口碑 |
| **运维能力要求** | 自建需要运维能力 | Docker Compose 极简部署，配置项最少化 |

## 六、市场机会

### 6.1 行业趋势

| 趋势 | 对 Nova 的影响 |
|------|--------------|
| 多模型并行成为标配 | 直接利好 — 核心能力 |
| 企业对 AI 成本敏感度上升 | 利好 — Cost Engine 是核心差异 |
| 数据安全合规要求增强 | 利好 — 私有部署 |
| AI 应用从单模型走向 Agent | 利好 — Agent Runtime 前瞻布局 |

### 6.2 目标市场份额

```
全球 AI Gateway 市场（2026 预测）
├── Portkey / OpenRouter / 云厂商 — 70%
├── 开源方案（One API / LiteLLM）— 20%
└── Nova AI Gateway 目标 — 10%
     └── 优先国内 + 东南亚中小团队
```

## 七、总结：Nova 的战略定位

```
不跟 Portkey 比 SaaS 规模
不跟 One API 比开源生态
跟它们比：

  ✅ 成本优化 — 竞品做不到
  ✅ 全栈能力 — 竞品不够深
  ✅ 私有部署 — 竞品不够轻
```

**一句话定位**：面向中小团队的自建 AI Gateway + Cost Engine 一体化平台。
